# Architecture

## Overview

familyTraffic is one Docker container with supervisord as PID 1, running xray, nginx, and certbot (plus optional mtg) on `network_mode: host`. nginx routes 443 by SNI, terminates TLS for proxies, and cloaks MTProxy. Host paths under `/opt/familytraffic` mount in; shell CLIs manage it.

## Single-Container Design

Since v5.33 the whole stack lives in one container named `familytraffic`, replacing the earlier multi-container layout.

It is built from a 3-stage Dockerfile (xray-src, mtg-src, final on `nginx:alpine` plus supervisor and certbot). `docker-compose.yml` declares a single service pulled from GHCR as `${GHCR_IMAGE}:${FT_IMAGE_TAG}`, restarted `unless-stopped`. Its healthcheck probes that TCP ports 8443 (xray Reality) and 443 (nginx stream) accept connections. See [[docker]].

## Process Supervision

supervisord (`/etc/familytraffic/supervisord.conf`) is PID 1 and orders startup by priority, since supervisord INI has no `depends_on`.

The order is `xray` (priority 1) → `nginx` (priority 2) → `certbot-cron` (priority 3). xray and nginx use `autorestart=true`; certbot-cron is a non-critical 12-hour renewal loop (`autorestart=false`). An `[include]` pulls optional programs from the host-mounted `/opt/familytraffic/config/supervisord.d/*.conf` — `mtg.conf` (created by `familytraffic-mtproxy setup`, removed by `disable`) enables the MTProxy process. See [[services]], [[mtproxy]], [[certificates]].

## Network Mode & Traffic Flow

With `network_mode: host` the container shares the host network stack — no Docker DNS, no inter-container networking. nginx fronts the public ports; xray binds loopback only.

Flow: `443` → nginx `ssl_preread` SNI routing — the default SNI passes through to `127.0.0.1:8443` (xray VLESS Reality, no TLS termination), while Tier 2 subdomains route to loopback `8448` (nginx http block) then to ws/xhttp/grpc inbounds on `8444`–`8446`. `1080` → nginx TLS termination → `127.0.0.1:10800` (SOCKS5). `8118` → nginx TLS termination → `127.0.0.1:18118` (HTTP proxy). `2053` → mtg (MTProxy Fake TLS) → Telegram. `4443` → nginx cloak-port for active-probing protection, loopback-only and never opened in UFW. Optional no-TLS ports `1081`/`8119` forward cleartext to the same proxy inbounds. See [[transports]], [[proxy-chains]], [[mtproxy]], [[security]].

## Host Paths

The installer creates `/opt/familytraffic` on the host and bind-mounts specific files (not directories, for configs) into the container.

`config/nginx/nginx.conf` (generated by `lib/nginx_stream_generator.sh`) mounts to `/etc/nginx/nginx.conf`; `config/xray_config.json` to `/etc/xray/config.json`; `data/users.json` to `/etc/xray/users.json`, live-reloadable via SIGHUP to xray. `config/mtproxy/` (mtg.toml + secrets.json) and `config/supervisord.d/` mount read-only; `logs/xray/` maps to `/var/log/xray` so host fail2ban can read it; `/etc/letsencrypt` and `/var/www/html` mount read-write for renewal. Runtime vars (`GHCR_IMAGE`, `FT_IMAGE_TAG`, `DOMAIN`, `ACME_EMAIL`) come from `/opt/familytraffic/.env`. See [[user-management]], [[certificates]], [[installation]].

## Entry Points

Installation runs `install.sh` as root; day-to-day management uses installed shell CLIs; the container itself starts via `entrypoint.sh`.

`install.sh` sources `lib/*`, detects the OS, installs dependencies, optionally acquires a Let's Encrypt certificate, then calls `orchestrate_installation` in `lib/orchestrator.sh` to build the directory tree, generate xray/nginx configs, write `docker-compose.yml` and `.env`, configure UFW, and deploy the container. The CLIs are `familytraffic` (users, transports, proxy, service ops), `familytraffic-mtproxy` (MTProxy lifecycle), `familytraffic-external-proxy` (upstream chains), and `familytraffic-cert-renew` (renewal hook). The container's `entrypoint.sh` does a first-run `certbot --standalone`, then `exec`s supervisord. See [[cli-tools]], [[installation]], [[testing]].
