# Transports

## Overview

familyTraffic runs one always-on Tier 1 transport, VLESS Reality on port 8443, plus optional Tier 2 transports (ws/xhttp/grpc) added via CLI. nginx handles port 443 with `ssl_preread` SNI routing, passing Reality through untouched and fanning Tier 2 subdomains to a loopback HTTP block. It also TLS-terminates SOCKS5 (1080) and HTTP (8118); xray routing steers per-user traffic to outbounds.

## VLESS Reality (Tier 1)

The primary transport is a single VLESS Reality inbound, tagged `vless-reality`, on internal port 8443 (`VLESS_PORT`) with `network: tcp` and `security: reality`. It is generated by `create_xray_config()` in `lib/orchestrator.sh`.

It is always present and never toggled by the transport CLI. Each client uses `flow: xtls-rprx-vision`. `realitySettings` carries `dest` (`REALITY_DEST:REALITY_DEST_PORT`), a `serverNames` list (the dest plus well-known sites like www.google.com), `privateKey`, and `shortIds` (per-user, schema v1.2, with `""` retained for compatibility). The inbound has a `fallbacks` entry to `127.0.0.1:80`. Clients reach it on the public port 443, which nginx forwards to 8443 (see [[architecture#Network Mode & Traffic Flow]]). User CRUD and shortId management live in [[user-management]].

## Tier 2 Transports

Tier 2 adds WebSocket, XHTTP (SplitHTTP), or gRPC as alternative camouflage transports, each on its own subdomain on port 443. State lives in `data/transports.json`; `lib/transport_manager.sh` exposes the lifecycle as CLI verbs (see [[cli-tools]]).

`add_transport <type> <subdomain>` validates the type (`ws`/`xhttp`/`grpc`) and the RFC 1123 subdomain, rejects a type already configured, then records `{type, subdomain, port, enabled}`. Internal xray ports are fixed: ws=8444, xhttp=8445, grpc=8446. The matching inbound (tag `vless-websocket`/`vless-xhttp`/`vless-grpc`) is surgically appended to `xray_config.json` with `jq` — `create_xray_config()` is deliberately NOT called, so existing users are preserved. nginx is regenerated, then `docker restart familytraffic` plus `nginx -s reload` apply changes. `remove_transport` reverses all of this by type. All JSON writes validate with `jq empty` and use `write_preserving_inode`.

## xray Inbounds and Stream Settings

Each transport maps to a distinct xray inbound with its own `streamSettings.network`: Reality uses `tcp`, ws uses `ws` (path `/vless-ws`), xhttp uses `splithttp` (path `/api/v2`), and grpc uses `grpc` (`serviceName` `GunService`).

Tier 2 inbounds are created with empty `clients: []` and `decryption: "none"`; users are shared into them by the user-management layer. A separate reverse-proxy HTTP inbound type exists in `lib/xray_http_inbound.sh`: `add_reverseproxy_inbound(domain)` allocates a loopback HTTP inbound (`listen: 127.0.0.1`, `protocol: http`) on the next free port in 10080–10089, tagged `reverse-proxy-<domain>`, used to chain traffic through upstream proxies (see [[proxy-chains]]). `reload_xray()` restarts the container to apply inbound changes.

## xray Routing

`lib/xray_routing_manager.sh` writes the xray `.routing` section that decides which outbound each flow uses. Base outbounds are `direct` (freedom) and `blocked` (blackhole); external proxy support adds `external-proxy` or per-user variants.

`domainStrategy` is `AsIs` throughout (chosen for mobile-network compatibility). `generate_routing_rules_json(mode)` emits rules for `all-traffic`, `selective`, or `disabled`. `enable_proxy_routing`/`disable_proxy_routing` swap the routing block and flip the `enabled` flag in `external_proxy.json`. For per-user chaining, `generate_per_user_routing_rules()` reads `users.json`, groups users by `external_proxy_id`, and emits one TCP rule per proxy (scoped by `inboundTag: ["vless-reality"]` and a `user[]` list), a UDP bypass rule sending UDP to `direct`, and a default rule routing everyone else to `direct`. `add_routing_rule(type, value, tag)` appends ad-hoc domain/ip/port/network rules. Changes require an xray restart (`supervisorctl restart xray`). Full chaining workflow is in [[proxy-chains]].

## nginx SNI Stream Routing

`generate_nginx_config` in `lib/nginx_stream_generator.sh` produces `nginx.conf`. Its `stream` block does L4 work: port 443 uses `ssl_preread` (reads SNI without decrypting TLS, keeping Reality intact) and proxies to `$backend_addr` from an SNI map.

The SNI map sends each configured Tier 2 subdomain to `127.0.0.1:8448` and everything else (`default`) to `127.0.0.1:8443` (Reality passthrough). Port 8448 lives in the `http` block: a `default_server` returns 444 for unknown hosts (active-probing protection), and per-subdomain `server` blocks TLS-terminate (TLSv1.3) and reverse-proxy to the matching xray port — `/vless-ws`→8444, `/api/v2`→8445, `/GunService/`→grpc 8446. The function takes flags for ws/xhttp/grpc subdomains plus optional MTProxy cloak-port 4443 (see [[mtproxy]]) and no-TLS ports 1081/8119.

## TLS Termination Ports

Beyond the 443 passthrough, the `stream` block TLS-terminates two proxy ports directly: 1080 (SOCKS5) → `127.0.0.1:10800` and 8118 (HTTP proxy) → `127.0.0.1:18118`, both TLSv1.3 with the Let's Encrypt cert (managed per [[certificates]]).

When `enable_notls` is set, plaintext mirrors are added: 1081→10800 and 8119→18118, transmitting credentials in cleartext (see [[security]]). All Tier 2 `http` server blocks and the cloak-port 4443 reuse the same `fullchain.pem`/`privkey.pem`. The `cert_domain` argument drives every certificate path, so a renewed certificate is picked up on the next nginx reload. Port 80 serves the ACME HTTP-01 challenge and redirects other requests to HTTPS.
